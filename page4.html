3.1 Review the benefits and drawbacks of IT tools and systems used in terms of productivity and efficiency



Being new to HTML and CSS, every little thing i tried to do,
whether it worked or not, always took time to do.
Time seemed to fritter away rather quickly trying to research hints and help.
If the subject had not been so personal then it would have been very beneficial working in a team.
Each member then could research different tasks and then pass on the information to each other.
With the use of a kanban board in Trello would have kept each member of the team up to date,
as it did as i worked solo on this project.





3.2 Describe ways to improve productivity and efficiency



With more practice and more time,
I could gain more proficiency using Visual Studio Code with HTML, CSS and Javascript.
With much more knowledge and understanding of the tools available to use on HTML and CSS,
I could do things much quicker and able to add so much more to the development of my webpage.
Again with more knowledge and practice of Javascript I could add much more function to my page.




3.3 Develop solutions to improve own productivity using IT in digital projects



Next time, having gained experience of this project, I certainly set myself realistic goals and time frames for each task.
Never be afraid to ask for help or advice!
I am still learning so I will not have all the answers or knowledge.
So ask and question and get as many opinions and answers as you can!
Next time I would certainly choose to work as part of a team.
Tasks can then be spread with the use of Trello.
Advice can then be sought by fellow team members via Slack.
It would then be easier to stick within time restraints.




3.4 Describe how you would go about testing digital solutions



To test a digital solution I would open the page and try to run each function,
checking that each function looks and does what it is supposed to do.
Then passing it on to colleagues to use and check to see if they can use and navigate through the page as I had planned,
without the bias I would more than likely have using my own design.
My colleagues can then see if there are any bugs that exist which I will not see with my own eyes.
